# Product Management

Learn how to create and manage products in Cartino's admin panel, from simple single-variant products to complex multi-option configurations.

[TOC]

## Products Overview

Navigate to **Products → All Products** to view your product catalog.

### Product List View

The product list shows:

- Product thumbnail
- Title and SKU
- Status (Active, Draft, Archived)
- Inventory count
- Price range
- Last updated date

### List Actions

- **Search**: Find products by title, SKU, or description
- **Filter**: By status, type, brand, category
- **Sort**: By date, title, price, inventory
- **Bulk Actions**: Update status, delete, export

## Creating a Product

### Simple Product (No Variants)

**Products → Add Product**

1. **Basic Information**
   - Title (required)
   - Description (rich text editor)
   - Product type: Physical, Digital, Service

2. **Pricing**
   - Price
   - Compare at price (for showing discounts)
   - Cost per item (for profit tracking)

3. **Inventory**
   - SKU (auto-generated or custom)
   - Barcode (EAN, UPC, etc.)
   - Track quantity: Yes/No
   - Quantity available
   - Allow backorders: Yes/No

4. **Organization**
   - Brand
   - Categories (multiple selection)
   - Collections
   - Tags

5. **Media**
   - Upload images (drag & drop)
   - Set featured image
   - Add videos
   - Reorder media

6. **SEO & URLs**
   - Page title (60 chars max)
   - Meta description (160 chars max)
   - URL handle (auto-generated from title)

7. **Save**
   - Save as draft
   - Save and publish

### Product with Variants

For products with options (size, color, material, etc.):

**Step 1: Add Options**

1. In the **Variants** section, click **Add Option**
2. Enter option name: "Size"
3. Add values: "S", "M", "L", "XL"
4. Add another option (optional): "Color"
5. Add values: "Red", "Blue", "Black"

**Step 2: Generate Variants**

1. Click **Generate Variants**
2. System creates all combinations (e.g., S-Red, S-Blue, M-Red, etc.)
3. Review generated variants

**Step 3: Configure Each Variant**

For each variant, set:

- SKU (auto-generated with pattern)
- Price (can differ per variant)
- Inventory quantity
- Weight and dimensions
- Barcode
- Images specific to this variant

**Example Configuration:**

```text
Product: Cotton T-Shirt
Options: Color (Red, Blue, Black) × Size (S, M, L)
Variants: 9 total

Variant 1: Red / S
- SKU: TS-RED-S
- Price: $19.99
- Inventory: 50

Variant 2: Red / M
- SKU: TS-RED-M
- Price: $19.99
- Inventory: 100

... and so on
```

## Product Variants

### Managing Variants

**Edit Product → Variants Section**

### Add Individual Variant

Click **Add Variant** to manually create:

1. Enter option values
2. Set SKU and price
3. Configure inventory
4. Upload variant images
5. Save variant

### Edit Variant

Click variant row to edit:

- Price and compare at price
- Inventory settings
- Physical properties (weight, dimensions)
- Images
- Barcode

### Reorder Variants

Drag and drop variants to change display order.

### Delete Variant

Click delete icon (⚠️ Cannot undo)

### Variant Pricing

Set prices per variant:

- **Base Price**: Default selling price
- **Compare At Price**: Show as strikethrough
- **Cost**: For profit calculations
- **Margin**: Auto-calculated

### Variant Inventory

For each variant:

- **Track quantity**: Enable/disable
- **Quantity**: Current stock level
- **Low stock threshold**: Alert level
- **Continue selling when out of stock**: Backorders

## Product Images

### Upload Images

**Product Edit → Media Section**

1. **Drag & Drop**
   - Drop files into upload area
   - Multiple files at once
   - Progress indicator

2. **Browse Files**
   - Click to browse
   - Select multiple files
   - Supported: JPG, PNG, WebP, GIF

### Image Management

- **Reorder**: Drag and drop
- **Featured Image**: Star icon
- **Alt Text**: For SEO and accessibility
- **Delete**: Remove image

### Variant Images

Assign specific images to variants:

1. Edit variant
2. Click **Assign Images**
3. Select from product images
4. Or upload new images

### Image Optimization

Images are automatically:

- Compressed for web
- Generated in multiple sizes
- Converted to WebP (with fallback)
- Lazy-loaded on frontend

### Recommended Sizes

- **Product Images**: 2048×2048px
- **Thumbnails**: Auto-generated (400×400px)
- **Gallery**: Auto-generated (800×800px)
- **Zoom**: Auto-generated (1600×1600px)

## Categories & Organization

### Categories

**Products → Categories**

Create hierarchical categories:

```text
Clothing
├─ Men's
│  ├─ T-Shirts
│  ├─ Shirts
│  └─ Pants
└─ Women's
   ├─ Dresses
   ├─ Tops
   └─ Skirts
```

**Creating Category:**

1. Click **Add Category**
2. Enter name and slug
3. Select parent (optional)
4. Add description
5. Upload category image
6. Set SEO fields
7. Save

### Brands

**Products → Brands**

Manage product brands:

1. Create brand
2. Add logo
3. Set description
4. Link to brand page

**Assign to Product:**

In product editor → Brand field → Select or create

### Collections

Group products for marketing:

**Products → Collections**

**Manual Collection:**

1. Create collection
2. Manually add products
3. Set featured products
4. Customize order

**Smart Collection:**

Auto-add products matching conditions:

```text
Collection: Summer Sale
Conditions:
- Tag contains "summer"
- Status is "active"
- Price < $50
```

### Tags

Free-form tags for products:

- Add in product editor
- Type to create new tag
- Autocomplete from existing
- Use for filtering and search

## Inventory Management

### Stock Tracking

**Enable/Disable per product:**

Product Edit → Inventory → Track quantity

**When enabled:**

- Inventory reduced on order
- Low stock alerts
- Out of stock prevention
- Stock history tracking

### Multi-Location Inventory

Manage stock across locations:

**Products → Inventory**

View inventory by:

- Product/Variant
- Location (Warehouse, Store, etc.)
- Availability status

**Adjust Stock:**

1. Find product variant
2. Select location
3. Enter new quantity
4. Add reason (optional)
5. Save adjustment

### Low Stock Alerts

**Settings → Inventory → Alerts**

Configure alerts:

- Low stock threshold (e.g., 10 units)
- Email notifications
- Dashboard alerts
- Per-product thresholds

### Stock Movements

View complete audit trail:

**Inventory → Stock Movements**

See all stock changes:

- Adjustments
- Sales
- Returns
- Transfers
- Corrections

**Each entry shows:**

- Date and time
- Product variant
- Location
- Quantity change
- Reason
- User who made change

## Pricing Strategies

### Base Pricing

Set on product or variant:

- **Price**: Main selling price
- **Compare at Price**: Original price (for discounts)
- **Cost**: Your cost (not shown to customers)

### Advanced Pricing

**Customer Group Pricing:**

Different prices for customer groups:

```text
Product: Cotton T-Shirt
Price: $19.99 (Regular)

Customer Group Pricing:
- Wholesale: $14.99
- VIP: $16.99
- Members: $17.99
```

**Site-Specific Pricing:**

Different prices per site/market:

```text
Product: Cotton T-Shirt

Site Pricing:
- US Store: $19.99 USD
- EU Store: €18.99 EUR
- UK Store: £16.99 GBP
```

**Volume Pricing:**

Quantity discounts:

```text
1-9 units: $19.99 each
10-49 units: $17.99 each (10% off)
50+ units: $14.99 each (25% off)
```

### Setting Up Pricing Rules

**Products → Pricing Rules**

1. Create pricing rule
2. Select products or categories
3. Set conditions (quantity, customer group, site)
4. Define price or discount
5. Set date range (optional)
6. Save rule

## Product Status

### Status Types

| Status | Description | Visibility |
|--------|-------------|------------|
| **Active** | Available for sale | Visible on storefront |
| **Draft** | Work in progress | Hidden from storefront |
| **Archived** | No longer sold | Hidden, kept for records |

### Publishing Workflow

1. Create product as **Draft**
2. Configure all settings
3. Add images and content
4. Review and test
5. Change status to **Active**
6. Set publish date (optional)

### Scheduled Publishing

**Product Edit → Status → Schedule**

- Set future publish date
- Auto-publish at specified time
- Email notification when published

## SEO Optimization

### SEO Fields

**Product Edit → SEO Section**

**Page Title:**

- 60 characters max
- Include primary keyword
- Brand name optional

**Meta Description:**

- 160 characters max
- Compelling description
- Include keywords naturally

**URL Handle:**

- Auto-generated from title
- Customizable
- Use hyphens, lowercase
- Include keywords

### Best Practices

**Good Example:**

```text
Product: Cotton T-Shirt - Red

Page Title: Red Cotton T-Shirt | Comfortable & Breathable | BrandName
Meta Description: Shop our premium red cotton t-shirt. Soft, breathable fabric perfect for everyday wear. Available in all sizes. Free shipping over $50.
URL: /products/red-cotton-t-shirt
```

**Image SEO:**

- Descriptive file names: `red-cotton-tshirt-front.jpg`
- Alt text for all images
- Optimized file sizes

## Product Duplication

Quickly create similar products:

1. Find product to duplicate
2. Click **Duplicate**
3. System creates copy with "(Copy)" suffix
4. Edit as needed
5. Save new product

**What's Copied:**

- Title (+ "Copy")
- Description
- Options and variants
- Categories and tags
- Custom fields

**What's NOT Copied:**

- SKUs (must be unique)
- Images (relink manually)
- Inventory quantities

## Tips & Best Practices

### Product Titles

- Clear and descriptive
- Include brand if relevant
- Add key features
- Keep under 70 characters

**Good:** "Nike Air Max 90 - Men's Running Shoes - White/Blue"

### Descriptions

- Focus on benefits, not just features
- Use bullet points for readability
- Include sizing information
- Add care instructions
- Mention unique selling points

### Images

- High-quality photos (minimum 1600px)
- Multiple angles (front, back, side, detail)
- Lifestyle images showing product in use
- Consistent background and lighting
- Show scale/size reference

### SKUs

Use meaningful pattern:

```text
Pattern: [Category]-[Brand]-[Product]-[Variant]

Examples:
TS-NIKE-AM90-WHT-10 (T-Shirt, Nike, Air Max 90, White, Size 10)
TS-GENERIC-001-RED-M (T-Shirt, Generic, 001, Red, Medium)
```

### Inventory

- Set low stock alerts
- Review regularly
- Use transfers for multi-location
- Track stock movements
- Perform regular audits
