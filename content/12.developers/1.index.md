```bash

# Install dependencies
```

## Addon Development Philosophy

Cartino's addon system is inspired by **Statamic CMS**:

### Key Principles

1. **Self-Contained** - Addons are complete packages
2. **Publishable** - Easy to share via Composer
3. **Configurable** - Expose settings to merchants
4. **Extensible** - Hook into core events
5. **Testable** - Include tests

### Addon Types

- **Feature Addons** - Add new functionality (SEO, Reviews, Subscriptions)
- **Integration Addons** - Connect external services (Mailchimp, Stripe)
- **Admin Addons** - Extend admin panel (Analytics, Reports)
- **Field Type Addons** - Custom blueprint fields

## Component Development Philosophy

### Admin Components

Admin UI components follow these principles:

1. **Composition API** - Modern Vue 3 patterns
2. **TypeScript** - Type safety
3. **Reka UI** - Headless, accessible base
4. **Tailwind CSS** - Consistent styling
5. **Reusable** - DRY components

### Example Component

```vue
<script setup lang="ts">
import { ref } from 'vue'
import { Button } from '@/Components/UI'

interface Props {
  title: string
  initialCount?: number
}

const props = withDefaults(defineProps<Props>(), {
  initialCount: 0
})

const count = ref(props.initialCount)

const increment = () => {
  count.value++
}
</script>

<template>
  <div class="card">
    <h2>{{ title }}</h2>
    <p>Count: {{ count }}</p>
    <Button @click="increment">Increment</Button>
  </div>
</template>
```

## API Development

### RESTful Conventions

```php
// routes/api.php
Route::prefix('v1')->group(function () {
    Route::apiResource('products', ProductController::class);
    Route::post('products/{product}/publish', [ProductController::class, 'publish']);
});
```

### Response Format

```php
// Success
return response()->json([
    'data' => $resource,
    'message' => 'Product created successfully',
], 201);

// Error
return response()->json([
    'message' => 'Validation failed',
    'errors' => $validator->errors(),
], 422);
```

## Testing Strategy

### Test Types

1. **Unit Tests** - Test individual methods
2. **Feature Tests** - Test HTTP endpoints
3. **Component Tests** - Test Vue components
4. **E2E Tests** - Test full workflows

### Example Tests

```php
// Unit Test
test('product calculates total correctly', function () {
    $product = Product::factory()->create(['price' => 1000]);

    expect($product->calculateTotal(2))->toBe(2000);
});

// Feature Test
test('admin can create product', function () {
    $admin = User::factory()->admin()->create();

    $this->actingAs($admin)
        ->postJson('/api/products', [
            'name' => 'Test Product',
            'price' => 1000,
        ])
        ->assertCreated();
});
```

## Contributing Guidelines

### Code Style

Follow PSR-12 for PHP and Vue.js style guide:

```php
// PHP
class ProductController extends Controller
{
    public function store(ProductRequest $request): JsonResponse
    {
        $product = Product::create($request->validated());

        return response()->json([
            'data' => new ProductResource($product),
        ], 201);
    }
}
```

```typescript
// TypeScript
interface Product {
  id: number
  name: string
  price: number
}

const fetchProducts = async (): Promise<Product[]> => {
  const response = await axios.get('/api/products')
  return response.data.data
}
```
