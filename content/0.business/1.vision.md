## Appendix A: Technical Architecture Deep Dive

### Database Schema Philosophy

Cartino usa un **hybrid approach**:
- **Relational tables** per dati strutturati (products, orders)
- **JSONB columns** per dati flessibili (custom fields)

```sql
-- Esempio: products table
CREATE TABLE products (
    id BIGINT PRIMARY KEY,
    name VARCHAR(255),
    slug VARCHAR(255),
    price INTEGER,
    -- ... colonne base
    fields JSONB, -- Custom fields da Blueprint
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- Index JSONB per performance
CREATE INDEX idx_products_fields ON products USING GIN (fields);
```
