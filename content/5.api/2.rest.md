# REST API Reference

Cartino's REST API provides access to all e-commerce functionality through standard HTTP methods and JSON payloads.

[TOC]

## Base URL

```text
https://your-store.com/api
```

## Request Format

All API requests should include:

```http
GET /api/products HTTP/1.1
Host: your-store.com
Authorization: Bearer YOUR_API_KEY
Accept: application/json
Content-Type: application/json
```

## Response Format

All responses return JSON:

```json
{
  "data": [...],
  "meta": {
    "current_page": 1,
    "per_page": 20,
    "total": 100
  }
}
```

## Products

### List Products

```http
GET /api/products
```

**Query Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `status` | string | Filter by status: `active`, `draft`, `archived` |
| `type` | string | Filter by type: `physical`, `digital`, `service` |
| `brand_id` | integer | Filter by brand ID |
| `search` | string | Search in title and description |
| `page` | integer | Page number (default: 1) |
| `per_page` | integer | Items per page (default: 20, max: 100) |

**Example Request:**

```bash
curl -X GET "https://your-store.com/api/products?status=active&per_page=10" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response:**

```json
{

        "id": 5,
        "name": "Example Brand"
      },
      "variants_count": 9,
      "lowest_price": 19.99,
      "highest_price": 29.99,
```

### Get Product

```http
GET /api/products/{id}
```

**Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `id` | integer | Product ID |
| `include` | string | Include relations: `variants`, `brand`, `categories`, `media` |

**Example:**

```bash
curl -X GET "https://your-store.com/api/products/1?include=variants,brand" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response:**

```json
{
  "data": {
    "id": 1,
    "title": "Cotton T-Shirt",
    "slug": "cotton-t-shirt",
    "description": "Comfortable cotton t-shirt",
    "type": "physical",
    "status": "active",
    "brand": {
      "id": 5,
      "name": "Example Brand"
    },
    "variants": [
      {
        "id": 1,
        "sku": "TS-RED-M",
        "title": "Cotton T-Shirt - Red / M",
```

### Create Product

```http
POST /api/products
```

**Request Body:**

```json
{
  "title": "New Product",
  "slug": "new-product",
  "description": "Product description",
  "type": "physical",
  "status": "draft",
  "brand_id": 5,
  "data": {
    "custom_field": "value"
  }
}
```

**Response:**

```json
{
  "data": {
    "id": 126,
    "title": "New Product",
    "slug": "new-product",
    "status": "draft",
    "created_at": "2024-12-09T10:00:00Z"
  }
}
```


## Orders

|-----------|------|-------------|
| `status` | string | Filter by status |
| `customer_id` | integer | Filter by customer |

| `to_date` | date | Orders to date (Y-m-d) |

## Filtering & Sorting

### Filtering

Use query parameters:

```http
GET /api/products?status=active&type=physical&brand_id=5
```

### Sorting

```http
GET /api/products?sort=-created_at
```

**Sort options:**

- `created_at` - Oldest first
- `-created_at` - Newest first
- `title` - A-Z
- `-title` - Z-A
- `price` - Low to high
- `-price` - High to low

### Search

```http
GET /api/products?search=cotton
```

## Including Relations

Use `include` parameter:

```http
GET /api/products/1?include=variants,brand,categories
```

**Multiple relations:**

```http
GET /api/orders/1?include=customer,items.variant,shipping_address
```

- `429` - Too Many Requests
